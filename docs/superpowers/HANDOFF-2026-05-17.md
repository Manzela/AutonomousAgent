## 2. Critical facts that are NOT discoverable from code

These are gotchas that cost real time to find. Don't re-discover them.

### 2.1 Gemini model id is `gemini-3.1-pro-preview` (with `-preview` suffix), only on `global` endpoint

- The plan originally said "Gemini 3.1 Pro" with model id `gemini-3.1-pro`. **Wrong.** The actual Vertex publisher model id is `gemini-3.1-pro-preview`.
- Probing the regional endpoint returns `404`. Only the `global` endpoint serves it:
  ```
  POST https://aiplatform.googleapis.com/v1/projects/i-for-ai/locations/global/publishers/google/models/gemini-3.1-pro-preview:generateContent
  ```
- The proxy entry in `deploy/litellm/config.yaml` already encodes this: `model_name: vertex_ai/gemini-3.1-pro-preview`, `vertex_location: global`.

### 2.2 Gemini 3.1 Pro Preview is a *thinking* model

- A 1-token answer ("pong") costs **~110-130 thought tokens internally** plus 1 text token.
- **Always set `max_tokens` ≥ 2048** when calling it. Lower values silently truncate the thoughts and return empty content. The smoke test (`scripts/smoke.sh` check 8) and the integration test (`tests/integration/test_p1_2_judge_panel.py`) already use 2048.

### 2.3 Branch protection on `main` rejects merge commits

- All PRs to `main` must use **squash merge** (`gh pr merge --squash`). Plain merge fails with `GraphQL: Merge commits are not allowed on this repository`.
- Required status checks: `Lint Python`, `Lint Shell`, `Lint YAML`, `Lint Dockerfiles`, `Unit Tests`, `Validate config/limits.yaml`, `Validate docker-compose`, `Conventional Commit title`, `Branch name follows convention`, `gitleaks`, `detect-secrets`.
- **Branch name regex** allows `session-<letter>/<path>` (added in PR #25). PR titles must be conventional-commit format (`feat:`, `fix:`, `chore:`, etc.).

### 2.4 ADC and gcloud are set up

- Active gcloud account: `manzela@tngshopper.com`, project `i-for-ai`.
- ADC works for Vertex AI calls.
