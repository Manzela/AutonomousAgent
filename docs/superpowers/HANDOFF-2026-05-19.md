## 7. Critical facts not discoverable from code

These are gotchas that cost real time to find. Don't re-discover them.

### 7.1 Hermes calls hooks as `invoke_hook(**kwargs)` — not positional
- Verified in `hermes-agent/hermes_cli/plugins.py:1253` + `:25` docs.
- The KEYSTONE bug (PR #56) was our `lib/durability/__init__.py` + `lib/durability/trichotomy.py` hooks declaring positional `(ctx, tool_call)`. Every invocation TypeError-ed silently at WARN level. A fresh session changing the hook signature contract must update **all durability submodules**: `__init__`, `failure_matrix`, `trichotomy`, `escalation`, `checkpoint`, `resume`.
- **Pattern for all new hooks**: mirror `lib/observability/__init__.py` — `(specific_kw=None, ..., **_: Any)` named-extraction + absorber.
- **Actual kwargs Hermes passes**:
  - `pre_tool_call`: `tool_name`, `args`, `task_id`, `session_id`, `tool_call_id`
  - `post_tool_call`: same + `result`, `duration_ms`. `result` is the tool's return value OR the caught Exception.
  - `on_session_start`: `session_id`, `model`, `platform`. **No `ctx`** — defensive `kwargs.get('ctx')` returns None.

### 7.2 Plugin loading mechanism (verified)
- Hermes' `PluginManager` (`hermes-agent/hermes_cli/plugins.py:747-905`) scans 4 sources in order: bundled (`<hermes_repo>/plugins/`), user (`~/.hermes/plugins/`), project (`./.hermes/plugins/` with env opt-in), pip entry points.
- Each plugin needs `plugin.yaml` manifest + `__init__.py` with `register(ctx)`.
- `plugins.enabled:` allow-list in `cli-config.yaml` is REQUIRED — discovery alone doesn't load.
- Our 6 plugins are mounted to `/root/.hermes/plugins/<name>` per PR #48 + path-rebased to `/home/hermes/.hermes/plugins/<name>` per PR #60 (non-root). The compose has *comments* referencing the legacy `/root/.hermes` paths (the PR #60 migration is documented inline at `deploy/docker-compose.yml:215-229`) but all *live mount paths* are `/home/hermes/.hermes`.

### 7.3 Container HOME is `/home/hermes`, not `/root` (post PR #60)
- `useradd hermes --create-home` set `HOME=/home/hermes`.
- All `Path.home() / ".hermes"` resolves to `/home/hermes/.hermes`.
- The `hermes-data` named volume is mounted to `/home/hermes/.hermes` (rebased from `/root/.hermes` per α-5 review).
- `volume-init` busybox sidecar runs as root, chowns to 1000:1000, exits 0. hermes service `depends_on: volume-init: service_completed_successfully`.
