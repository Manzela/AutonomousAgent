PLACEHOLDER — user must complete this step before cron pings will work.

Healthchecks.io URL is a per-user secret. The plan defers it to a manual user
step (Phase-1 plan T30 step 1). To complete the setup once you have the URL:

1. Sign up at https://healthchecks.io (free tier is sufficient).
2. Create a project named `hermes-local` and copy the unique ping URL,
   e.g. https://hc-ping.com/<YOUR-UUID>.
3. Encrypt and store it:

       echo "https://hc-ping.com/<YOUR-UUID>" > secrets/healthchecks-url
       sops -e secrets/healthchecks-url > secrets/healthchecks-url.sops
       rm secrets/healthchecks-url

4. Verify it decrypts:

       SOPS_AGE_KEY_FILE=$HOME/.config/sops/age/keys.txt \
         sops -d secrets/healthchecks-url.sops

5. Commit the encrypted file:

       git add secrets/healthchecks-url.sops
       git commit -m "feat(observability): add encrypted Healthchecks.io URL"

6. Delete this template file:

       git rm secrets/healthchecks-url.template.txt
       git commit -m "chore(secrets): remove healthchecks-url template (real URL is in place)"

Until step 3 is complete, scripts/healthcheck-ping.sh will exit non-zero with
a clear error message rather than silently failing.
